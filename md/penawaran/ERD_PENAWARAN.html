Catatan Teknis

⚠️ Penting untuk Developer

  • Denormalisasi — Field nama (nm_customer, nm_divisi, dll) disimpan langsung di tabel transaksi untuk menghindari JOIN berlebihan saat reporting.
  • Soft Delete — Pola deleted_by + deleted_date. NULL = aktif, terisi = dihapus.
  • History/Audit Trail — Setiap revisi menghasilkan snapshot lengkap di tabel _history dengan ID format HST-YYYYMM-XXXXX.
  • Dual Database — Data employee dan divisi dari database hr_sentral (koneksi dbhr), bukan dari db_consultant_new.
  • Budget vs Penawaran — Tabel others, lab, subcont_tenaga_ahli, dan subcont_perusahaan memiliki dual pricing: price_unit (ke customer) dan price_unit_budget (budget internal).
  • Approval — Dikendalikan oleh sts_quot (NULL=draft, P=pending, A=approved), sts_reject, dan sts_deal.
