Catatan Teknis

⚠ Penting untuk Developer

  • Relasi Transaksional: Semua tabel detail (akomodasi, lab, others, subcont, dll) terhubung ke tabel kons_tr_kasbon_project_header menggunakan field id_header.
  • Pola Pengajuan vs Estimasi vs Terpakai: Setiap baris di tabel detail tidak hanya menyimpan nilai pengajuan kasbon, tetapi juga menduplikasi sisa budget dari SPK Budgeting.
  • Data Denormalisasi: ID referensi tetap disimpan pada tabel detail untuk mempermudah Query tanpa JOIN ke header.
